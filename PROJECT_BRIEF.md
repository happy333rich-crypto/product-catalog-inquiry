# 產品電子型錄＋詢價清單：第一版需求

## 專案目標
建立一個以手機操作為優先的繁體中文產品型錄網站，讓客戶可以瀏覽商品、加入詢價清單、填寫數量與聯絡資料，最後產生一段可直接複製到 LINE 的詢價文字。

第一版只做前端操作雛形，不接資料庫、不串 Notion、不寄 Email、不呼叫外部 API，也不自動決定價格。

## 技術限制
- 使用原生 HTML、CSS、JavaScript
- 不使用 React、Vue 或需要安裝的套件
- 以手機版為優先，也要能在 iPad 與電腦正常使用
- 商品資料從 `products.json` 載入
- 詢價清單與顧客填寫內容使用 `localStorage` 暫存
- 所有文字使用繁體中文

## 資料安全規則
- 不顯示任何進價、底價、毛利率、促銷底價或內部備註
- 不在程式裡放帳號、密碼、Token、API Key
- 第一版不把顧客資料送到伺服器
- 商品價格先完全不顯示

## 商品型錄功能
1. 首頁顯示 `products.json` 中 `active: true` 的商品。
2. 商品卡片顯示：商品圖片預留框、品牌、商品名稱、規格、箱入數、貨號、加入詢價按鈕。
3. 可以用商品名稱、貨號或條碼搜尋。
4. 可以依品牌篩選。
5. 可以依商品分類篩選。
6. 同一商品重複加入時，不要新增重複項目，改為增加數量或提示已加入。

## 詢價清單功能
1. 畫面上要有明顯的「詢價清單」入口與已加入商品數量。
2. 每項商品可設定詢價數量，單位預設為「箱」。
3. 可用加號、減號調整數量。
4. 可刪除單一商品。
5. 可一鍵清空整份詢價清單，但要先確認。
6. 重新整理網頁後，詢價清單不要消失。

## 顧客資料欄位
- 店家名稱（必填）
- 聯絡人（必填）
- 聯絡電話（必填）
- 縣市／地區
- 備註

## 產生詢價內容
按下「產生詢價內容」後，產生：

```text
【產品詢價】
店家名稱：
聯絡人：
聯絡電話：
地區：

詢價商品：
1. 商品名稱｜規格｜數量 X 箱
2. 商品名稱｜規格｜數量 X 箱

備註：
```

並提供「一鍵複製詢價內容」按鈕；複製成功後顯示提示。若必填資料未填或詢價清單為空，要顯示白話提示。

## 介面與視覺
- 整體乾淨、清楚，不要像工程師測試頁面
- 按鈕要大，方便手機點擊
- 商品卡片資訊要有層次
- 主色可用柔和綠 `#99B9A3`
- 強調按鈕可用暖橘 `#F4A688`
- 手機版詢價清單入口要容易找到，可固定在底部
- 手機、iPad、電腦都不能水平溢出

## 建議檔案結構
- `index.html`
- `style.css`
- `script.js`
- `products.json`
- `README.md`

## 第一版不做
- 不收集或上傳顧客資料
- 不串接 Notion、Google Sheets、Email、LINE API
- 不登入會員
- 不顯示任何價格
- 不產生正式報價單 PDF
- 不做自動報價
- 不補商品圖片

## 驗收項目
1. 18項商品都能正常顯示。
2. 搜尋商品名稱、貨號、條碼皆有效。
3. 品牌與分類篩選正常。
4. 加入詢價、加減數量、刪除商品正常。
5. 重新整理後，詢價清單仍存在。
6. 顧客資料必填檢查正常。
7. 能產生正確的繁體中文詢價文字。
8. 一鍵複製可用；不支援時要有替代提示。
9. 畫面在手機、iPad、電腦都不會水平溢出。
10. 不得在任何頁面或程式碼中出現進價、毛利率或內部報價資料。
